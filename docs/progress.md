# 개발 진행 상황 정리 (v2.0 - AI App with MLOps)

**전략 선회:** 기존 '하드웨어 스캐너'에서 'AI + 바코드 스마트폰 앱'으로 계획을 변경했으며, 유료 API 의존성을 줄이고 오픈소스 솔루션(PaddleOCR, Open Food Facts)을 적극 활용합니다. 또한, 사용자 피드백 기반의 AI 모델 개선을 위한 MLOps 파이프라인 구축을 목표로 합니다.

---

## ✅ 완료된 작업

*   **프로젝트 기획 및 재설계**
    *   `[✅]` `log.md`를 통해 PaddleOCR, 하이브리드 바코드 조회, MLOps, Expo, 개발 클라이언트 등 새로운 기술 스택 및 아키텍처 확정
    *   `[✅]` 모든 관련 문서 (`README.md`, `weekplan.md`, `frontend_plan.md` 등) 최신화 완료
*   **백엔드 환경 및 핵심 기능 구현**
    *   `[✅]` `requirements.txt`에 신규 라이브러리 추가 및 DB 스키마 최신화
    *   `[✅]` **OCR 엔진 교체:** Clova OCR 로직을 **PaddleOCR** 기반으로 성공적으로 교체
    *   `[✅]` **`/upload_receipt` API 개선:** 새로운 DB 스키마(`receipts`, `receipt_items`)에 맞게 저장 로직 수정 및 `category_id` 매핑 로직 추가
    *   `[✅]` **`/lookup_barcode` API 구현:** 3단계 하이브리드 조회 전략(Open Food Facts -> FOOD_QR -> 식품안전나라), 예외 처리, 카테고리 매핑 로직 적용 완료
*   **프론트엔드 초기 설정 (Phase 1)**
    *   `[✅]` **Expo 프로젝트 생성** 및 핵심 라이브러리 설치 완료
    *   `[✅]` **개발 클라이언트 설정:** EAS 빌드를 통해 네이티브 라이브러리 사용 환경 구축 완료
    *   `[✅]` **Expo Router 기반 화면 구조 설정** 및 하단 탭 네비게이션 구현 완료
    *   `[✅]` **카메라 권한 설정 및 뷰 렌더링** 완료
*   **프론트엔드 핵심 기능 (Phase 2 & 4 일부)**
    *   `[✅]` **바코드 스캔을 통한 빠른 상품 등록 기능 완료:** 바코드 인식, 3단계 하이브리드 API 조회, 카테고리 매핑, 유통기한 입력 및 Supabase `inventory` 테이블 저장까지 전체 흐름 구현 완료
    *   `[✅]` **Supabase 클라이언트 설정:** 앱 전역에서 Supabase에 접근할 수 있도록 클라이언트 설정 완료
